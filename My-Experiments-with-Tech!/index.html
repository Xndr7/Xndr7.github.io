My Experiments with Tech!-Part 0

Blog Initializing.........

Hey guys! In this series of posts, I’m going to focus on a few of the various tech-related experiments that I tend to perform.

In this particular post, since it’s the first one I’m going to focus on Blogging and and more specifically the process of setting up a Blog using Middleman and Github for hosting.

What is a Blog?

A blog can be very specifically defined as a regularly updated website or web page, typically one run by an individual or small group, that is written in an informal or conversational style. Blogs are created for a variety of purposes like tech news, education,fashion, gossip, etc. Some famous blogs are TMZ and TechCrunch.

How to create a blog using Github and Middleman

Github is one of the biggest repository hosting services that offers version control as well as few of it’s own features. One such feature is that every user can make a static site and serve it through Github Pages. Create an account before continuing with the process.

Middleman is a static site generator that uses ruby to quickly generate and deploy these sites through Github Pages.

Steps to install Middleman:

$gem install middleman
Successfully installed middleman-3.3.7
1 gem installed

Also, since we want to make a blog, we can install the middleman-blog gem as well as use the middleman-deploy gem to deploy the site.

$gem install middleman-blog
$gem install middleman-deploy

Ruby will be needed to be installed first.

On Windows, goto RubyInstaller and select the latest version(32-bit or 64-bit) and install it.

Then, from the same website, download the Ruby Dev-kit and extract it using 7z. Goto this page for more help on the installation process.

On Unix, install the build-essentials and ruby-dev packages.

If you are using any forked repo from github all that has to be done is:

$sudo gem install bundler
$bundle install

This process uses the Gemfile in the Repo to automatically fetch the required gems and makes the process simpler and easy to follow. Few middleman based Github Repos are:

  • Casper
  • Bootstrap based template
  • This site

To create a default middleman based blog we can use the middleman-blog gem

$middleman init {Github_Username}.github.io --template=blog

Replace {Github_Username} with your own Github Username. This step creates all the files necessary to generate a blog and also downloads the required gem dependencies.

The Middleman Server

Middleman has the feature of running a server on your local machine. This let’s us see a functioning preview of our blog on creation and for prototyping purposes.

$cd {Github_Username}.github.io
$middleman server

Goto http://localhost:4567 or its local equivalent to see the site. If any errors are detected, solve them first. Most errors are caused due to Build tools and dev-kit being improperly installed. Follow the instructions above and it should work well.

If everything goes well, you will see an example article on the local site.

Creating a new article

You can create an article using markdown and place it in the related folder. Names of the files follow the "{%year%}-{%month%}-{%day%}-{%title%}.md format.

You can create a new article using middleman article. In the project root folder, run the following:

$middleman article "This is a test post"

Refresh your local site to see the changes made.

Deploying your site!

If you have reached here, step back and have some coffee and take a break! You have earned it!

To continue, we need to configure the setup to deploy to our Github repo directly.This is done by changing the config.rb file in the root directory.

Add the following lines to this file:

activate :deploy do |deploy|
  deploy.method = :git
  deploy.branch = 'master'
  deploy.build_before = true
end

This will deploy the site to Github and use the “master” branch and remote=origin to do so. These configurations can be done when setting Git up.

Git setup

In the root directory,

$git init
$git checkout -b change-branch 
$git commit -m "Init"
$git push --set-upstream origin change-branch
$git remote add origin https://github.com/{Github_Username}/{Github_Username}.github.io.git

We wont do git push -u origin master as that is what middleman deploy will do.

Before executing the last command, create a new repository on Github with the name specifically in the following format: {Github_Username}.github.io

This ensures that your site is created properly and linked to the correct repo.

Now run the following command:

$middleman deploy

You have to enter your Github username and password and it will deploy it to the proper branch. After few moments check {Github_Username}.github.io and you will see your site live.

Adding articles to Github

This can be done in 3 simple steps:

  • Create a new post using middleman article "TITLE"
  • Write your post
  • Deploy to Github using middleman deploy

And you’re done!

If you want to add a specific domain name create a file CNAME in the source branch and add the name with the “https://www” to it.

Go Forth and Create!!

Hopefully, you now realize that Middleman along with GitHub is an extremely potent and cost-effective combination to create static websites. Middleman also has a pleasant workflow. Extensions are available either in the form of Middleman extensions, or as Ruby gems.

This post didn’t cover customising the look and feel of the site. That’s because it is no different from developing and styling any other site! For inspiration, you can look at other sites that were developed using Middleman. Thanks for reading!
